## 5. מבנה MDX – Frontmatter

כל מאמר/טיפ הוא קובץ `.mdx` בתיקיית `/content/posts/` עם frontmatter:

```yaml
---
title: "כותרת המאמר"
date: "2025-02-20"
tags: ["AI", "קהילות", "טכנולוגיה"]
type: "article"   # או "tip"
published: true   # false = לא מופיע באתר
slug: "my-post-url"
---
```

- `published: false` – הקובץ קיים בגיטהאב אבל לא מופיע בשום מקום באתר
- `type: "tip"` – מוצג עם תגית "טיפ" בכרטיס
- `title` מהפרונטמטר מופיע ככרטיס בדף הבלוג (אין תיאור נפרד)

## 9. ממשק ניהול (Admin)

### 9.1 התחברות (`/admin/login`)

- שדות: אימייל + סיסמה
- סיסמה מאוחסנת כ-bcrypt hash ב-environment variable
- Session ב-httpOnly cookie (תוקף: 7 ימים)
- אחרי התחברות → redirect ל-`/admin`

### 9.2 ממשק כתיבה (`/admin`)

**שדות:**

| שדה | תיאור |
|-----|-------|
| כותרת | כותרת המאמר (תופיע בכרטיס בבלוג) |
| slug | סיומת URL (לדוגמה: `my-post`) |
| תגיות | input עם תגיות מופרדות בפסיק |
| סוג | dropdown: מאמר / טיפ |
| תאריך | date picker (ברירת מחדל: היום) |
| מצב | toggle: פורסם / טיוטה |
| תוכן | textarea גדולה לכתיבת Markdown |
| העלאת קובץ | drag & drop / בחירת קובץ MD או TXT מהמחשב |

**תצוגה מקדימה:**
- לחצן "תצוגה מקדימה" שמציג את המאמר מרונדר בצד

**שמירה:**
- לחצן "שמור ופרסם" / "שמור כטיוטה"
- דוחף קובץ MDX לגיטהאב דרך GitHub API
- שם הקובץ: `{slug}.mdx`

## 12. הערות חשובות

1. **אין WordPress** – פרויקט Next.js נקי לחלוטין
2. **אין Firebase** – תוכן MDX ב-Git בלבד
3. **הכל RTL** – לבדוק כל component שה-padding/border/alignment נכונים
4. **לוגיקת BiDi לקוד** – קוד תמיד LTR גם בתוך תוכן RTL
5. **published: false** – מאמרים בגיטהאב שלא מופיעים באתר בכלל
6. **Admin מוגן** – httpOnly cookie, לא localStorage
7. **שאלות ותשובות** – accordion בכל עמוד תוסף, לא הכל פתוח
8. **טבלאות** – תמיד עטופות ב-`.table-responsive-wrapper` לרספונסיביות
